# 研究设计文档
## AI生成图像识别：策略培训对检测准确性的影响

---

## 1. 研究背景

随着AI图像生成技术（Midjourney、DALL-E、Stable Diffusion等）的快速发展，AI合成图像与真实照片在视觉上已高度相似。普通用户在日常信息消费中越来越难以区分内容的真实性，这对媒体素养、错误信息传播及公众认知构成挑战。

已有研究表明，人们在未经训练的情况下识别AI图像的表现接近随机水平（Nightingale & Farid, 2022）。尽管如此，部分研究提示，提供明确的感知线索（如解剖异常、纹理不一致）有助于提升识别准确率。本研究在此背景下，检验**结构化策略培训是否能显著改善普通用户的AI图像识别能力**，并探究外部知识工具（反向图片搜索）的使用行为。

---

## 2. 研究问题

**RQ1（主要）**：接受AI图像识别策略培训的参与者（组C），其识别准确率是否显著高于仅接受AI图像背景知识的对照组（组A）？

**RQ2**：策略培训是否会影响参与者的判断信心？

**RQ3**：策略培训是否会促进参与者主动使用外部知识工具（模拟Google Lens反向搜索）？

**RQ4**：参与者自评的AI图像辨别能力（自我效能感）在实验前后是否发生变化，且变化是否因实验组别而有所不同？

**RQ5**：哪些个体差异变量（AI熟悉度、使用频率、教育程度等）与识别准确率相关，是否调节策略培训的效果？

---

## 3. 研究假设

| 编号 | 假设 | 方向 |
|------|------|------|
| **H1** | 组C参与者的图像识别准确率显著高于组A | C > A |
| **H2** | 组C参与者对其判断的平均信心显著高于组A | C > A |
| **H3** | 组C参与者使用Google Lens搜索功能的比例和频次显著高于组A | C > A |
| **H4** | 实验后自评能力（post_self_efficacy）高于实验前（self_assessed_ability），且组C的提升幅度大于组A | C变化量 > A变化量 |
| **H5** | 较高的AI熟悉度（ai_familiarity）与更高的识别准确率正相关 | 正相关 |
| **H6（探索性）** | AI熟悉度对策略培训效果起调节作用：低熟悉度参与者从培训中获益更多 | 交互效应 |

---

## 4. 研究设计

**设计类型**：单因素被试间设计（One-Factor Between-Subjects）

**自变量**：实验组别（Group）
- **组A（控制组）**：阅读关于AI图像生成技术的背景介绍材料（无识别策略）
- **组C（策略组）**：阅读包含三种识别策略的结构化培训材料

**主要因变量**：
- 图像识别准确率（accuracy）：正确判断图像真实性的比例
- 信号检测理论指标（d'、criterion c）：灵敏度与响应偏向

**次要因变量**：
- 判断信心（confidence）：每张图片判断的自信程度（1-5分）
- Google Lens使用行为（lens_open_count等）
- 自我效能变化（post_self_efficacy - self_assessed_ability）
- 反应时（response_time_ms）

**随机分配**：区组随机化（block size=2），按当前各组已分配人数动态平衡，保证组A与组C的参与者数量始终均衡。

---

## 5. 参与者

**招募方式**：在线实验，通过链接传播。

**纳入标准**：
- 年龄18岁及以上
- 自愿参与，同意知情同意书

**排除标准**（数据分析阶段）：
- 未完成全部实验阶段（`completed = false`）
- 未通过注意力检验（`attention_check_answer ≠ 5`）
- 属于历史测试的组B数据（分析时仅保留A、C组）

**目标样本量**：根据先前研究估计效应量 d=0.4（中等偏小），α=.05，1-β=.80，每组约需 100 名参与者（共约 200 名）。

---

## 6. 实验材料

### 6.1 图像刺激池

共 **40 张图像**：
- AI生成图像：20 张，来源涵盖 ai-art、Midjourney、Nanobanana
- 真实图像：20 张，来源涵盖拍摄照片（camera）和网络图片（website）

图像风格分布（AI与真实图像均包含）：

| 风格 | 说明 |
|------|------|
| `illustration` | 插画/绘画风格 |
| `photograph` | 写实照片风格 |
| `cartoon` | 卡通/动漫风格 |

每位参与者看到的图像顺序通过其 `participant_id` 作为随机种子（`image_seed`）进行确定性打乱（Fisher-Yates shuffle），保证结果可复现且不同参与者之间顺序不同。

实验配置参数（`experimentConfig.ts`）：
- `totalImages`：实际展示的图片总数（需为偶数，AI:Real=1:1）
- 当前配置值为 **2**（测试阶段），正式实验建议调整为 **40**

### 6.2 干预材料

**组A：《关于AI生成图像》**（背景知识，控制条件）

包含以下内容，但**不提供**任何识别策略：
- AI图像生成技术的发展历程
- AI图像生成的技术原理（GAN、扩散模型）
- AI图像的应用领域（广告、艺术等）

**组C：《识别AI生成图像的策略》**（策略培训，实验条件）

包含三种结构化识别策略，每种策略附有示例图：

| 策略 | 核心内容 | 示例图 |
|------|----------|--------|
| 策略一：检查人体解剖细节 | 手指数量异常、手部姿势不自然、面部特征不对称、牙齿排列异常等 | anatomy.png |
| 策略二：检查风格和纹理 | 纹理不一致、过度光滑的塑料质感、整体"感觉"异常 | style.png |
| 策略三：利用外部知识 | 反向图片搜索（Google Lens模拟功能）判断图像来源 | knowledge.png |

两组参与者在进入图像判断任务前**至少阅读3秒**（`interventionMinStaySeconds=3`），系统记录实际阅读时长（`intervention_duration_s`）。

### 6.3 Google Lens模拟功能

实验内置模拟的Google Lens反向搜索功能：
- **触发方式**：在图像上右键点击（桌面端）或长按600ms（移动端）
- **操作**：在弹出菜单中选择"使用Google搜索图片"
- **展示内容**：预设的模拟搜索结果（来自 `public/data/lens-results.json`），每张图展示6条结果
- **系统记录**：所有交互行为均写入数据库（见§8.4）

## 7. 实验流程

参与者按以下顺序完成6个阶段：

```
阶段0：欢迎页 & 知情同意
  ├─ 展示研究概述、流程、隐私政策
  ├─ 参与者选择"同意并继续"（或"不同意"退出）
  └─ 系统创建参与者记录，分配组别（A或C），生成随机种子

阶段1：前测问卷（背景信息）
  └─ 收集人口统计信息 + AI使用经验基线

阶段2：干预材料阅读
  ├─ 组A：AI背景知识
  ├─ 组C：三种识别策略
  └─ 计时开始，记录阅读时长（intervention_duration_s）

阶段3：图像判断任务
  ├─ 逐张展示图像，参与者判断"AI生成"或"真实照片"
  ├─ 每张记录：判断结果、信心评分（1-5）、可选文字理由、反应时
  └─ 可使用Google Lens模拟搜索功能，所有交互行为被记录

阶段4：后测问卷
  └─ 操纵检验、策略使用程度、自我效能后测、注意力检验

阶段5：结束页
  └─ 展示个人成绩（答对X/Y张），标记完成
```

## 8. 测量工具与变量说明

### 8.1 前测问卷变量

| 字段名 | 类型 | 尺度 | 内容 |
|--------|------|------|------|
| `age` | 分类 | 5级 | 年龄段：18-24, 25-34, 35-44, 45-54, 55+ |
| `gender` | 分类 | 3级 | 性别：男、女、不愿透露 |
| `education` | 分类（有序） | 5级 | 最高学历：高中、大专、本科、硕士、博士及以上 |
| `ai_tool_usage` | 二分 | 是/否 | 是否使用过AI图像生成工具 |
| `ai_familiarity` | 连续 | 1-5 Likert | 对AI生成图像的熟悉程度 |
| `self_assessed_ability` | 连续 | 1-5 Likert | 自评区分AI/真实图像的能力（实验前基线） |
| `ai_exposure_freq` | 有序 | 5级 | 日常接触AI生成内容的频率 |

### 8.2 图像判断任务变量（每张图片一条记录）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `image_id` | 字符串 | 图像唯一标识（如 `ai_03`、`real_08`） |
| `image_order` | 整数 | 该图在本次任务中的展示顺序（1-indexed） |
| `judgment` | 分类 | 参与者判断：`AI` 或 `Real` |
| `correct_answer` | 分类 | 图像真实标签：`AI` 或 `Real` |
| `is_correct` | 布尔 | 判断是否正确 |
| `confidence` | 整数 | 信心评分，1-5分（1=完全猜测，5=非常确定） |
| `reasoning` | 文本 | 判断理由（可选，最长200字符） |
| `response_time_ms` | 整数 | 从图像加载完成到提交判断的时间（毫秒） |

### 8.3 后测问卷变量

| 字段名 | 适用组别 | 类型 | 说明 |
|--------|----------|------|------|
| `manipulation_check_read` | 全部 | 单选 | 是否阅读了关于识别策略的材料（是/否/不确定） |
| `manipulation_check_strategies` | 仅C组 | 单选（是/否） | 材料中是否提到"检查风格与纹理"策略——操纵检验核心题目 |
| `strategy_usage_degree` | 仅C组 | 1-5 Likert | 在做判断时使用策略的程度（1=完全没有，5=大量使用） |
| `open_method` | 全部 | 开放文本 | 主要判断方法的自由描述（最长500字符） |
| `self_performance` | 全部 | 1-5 Likert | 自评实验表现（1=非常差，5=非常好） |
| `post_self_efficacy` | 全部 | 1-5 Likert | 实验后自评AI识别能力（1=非常差，5=非常好） |
| `attention_check_answer` | 全部 | 1-5 Likert | 注意力检验题（要求选5） |
| `attention_check_passed` | 全部 | 布尔 | 注意力检验是否通过（=选5则true） |

### 8.4 交互行为日志

系统记录参与者与Google Lens功能的全部交互事件：

| 事件类型 | 说明 |
|----------|------|
| `TRIGGER_MENU` | 右键点击/长按触发上下文菜单 |
| `OPEN_LENS` | 点击"使用Google搜索图片"打开搜索面板 |
| `SCROLL_LENS` | 在搜索结果面板中滚动（触发阈值：50px） |
| `CLICK_RESULT` | 点击某条搜索结果 |

每条日志记录：`participant_id`、`image_id`、`image_order`、`action`、`client_timestamp`

## 9. 数据收集与存储

**数据库**：MongoDB（通过 Mongoose）

**集合结构**：

| 集合名 | 对应模型 | 关键索引 |
|--------|----------|----------|
| `participants` | Participant | `participant_id`（唯一）, `prolific_id`（稀疏） |
| `responses` | Response | `{participant_id, image_id}`（联合唯一） |
| `postsurveys` | PostSurvey | `participant_id`（唯一） |
| `interactionlogs` | InteractionLog | `participant_id` |

**数据导出**：通过 `analysis/fetch_data.py` 脚本从MongoDB下载为CSV文件，存放于 `analysis/data/` 目录。

---

## 10. 数据质量控制

| 控制措施 | 实现方式 |
|----------|----------|
| 注意力检验 | 后测问卷设陷阱题，要求参与者选"5（非常同意）"；未通过者从分析中排除 |
| 操纵检验 | 询问组C参与者是否记得"检查风格与纹理"策略；未通过者标记为操纵检验失败（可敏感性分析） |
| 最短阅读时间 | 干预材料页设3秒倒计时锁，确保参与者至少看到材料 |
| 唯一响应约束 | 数据库层面保证每位参与者对每张图片只能提交一次判断 |
| 完成状态过滤 | 仅分析 `completed=true` 的参与者数据 |
| 会话恢复 | 使用 sessionStorage 保存进度，防止意外刷新导致数据丢失 |

## 附录：关键技术参数

| 参数 | 值 | 说明 |
|------|----|------|
| `totalImages` | 40（正式）/ 2（测试） | 每位参与者看到的图片数量 |
| `interventionMinStaySeconds` | 3秒 | 干预材料最短停留时间 |
| `image_seed` | = participant_id | 图像顺序随机种子（Fisher-Yates） |
| `confidence scale` | 1-5 Likert | 判断信心评分范围 |
| `longPressThreshold` | 600ms | 移动端触发上下文菜单的长按时长 |
| `scrollLogThreshold` | 50px | 记录搜索结果滚动行为的最小滚动距离 |
| `resultsPerImage` | 6条 | 每张图片的模拟搜索结果数量 |
